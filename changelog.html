Release Notes

FillPro Changelog

A short release log for what shipped, what changed, and the limits users should know about.

Latest release

Version 1.0.0

April 26, 2026 | Launch edition

The first public FillPro release focuses on reusable profiles, local-first storage, smart rules, local upload matching, and clear unsupported states.

What shipped

Core product flow

  • One-click Fill Page from the popup for normal accessible web forms.
  • On-page profile picker so filling can start without breaking focus.
  • Custom fields and smart rules for awkward labels and site-specific matching.
  • Local file upload matching for resume, CV, and cover-letter style inputs.

Trust posture

Privacy and scope

  • Profile data stays local to the extension on the device.
  • No broad production host permissions.
  • Sensitive fields are intentionally skipped.
  • Optional feedback diagnostics are user-triggered and prepared locally.

User experience

First-run experience

  • A lighter first-run path that pushes toward a real fill instead of a long tutorial.
  • Support pages written for practical troubleshooting and privacy guidance.
  • Free mode with no account required for core use.

Known boundary

Known browser and site limits.

FillPro does not bypass CAPTCHA, anti-bot systems, blocked login walls, cross-origin payment frames, or browser-native PDF viewers. If a page blocks access before a normal form is reachable, the extension reports that limit.
